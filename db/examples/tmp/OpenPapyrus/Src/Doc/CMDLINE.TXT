
Командная строка программы Project Papyrus
--------------------------------------------------------------------------------
PPW [user] /option:param

user:
	Имя пользователя для входа в систему.
	Если этот параметр равен recover, то запускается процедура
	восстановления базы данных.

Опции:
	/x-
		Запрещает использование расширенной памяти для загрузки
		оверлейных сегментов.
		Единственный известный мне случай, когда эта опция необходима -
		запуск программы под отладчиком Borland C++ 3.1 в режиме
		эмуляции DOS из Windows95.
	/l-     *** В версии 3.0.11 упразднен
		Запрещает использование логических блокировок Novel NetWare
		для блокировки объектов. Этот параметр может понадобиться в
		случае, когда без всяких разумных оснований выдается сообщение
		"Ошибка логической блокировки".
	/cash:cashNodeName
		Запуск в режиме просмотра и ввода чеков синхронного кассового
		узла cashNodeName
	/prc:processorName или /prc:processorID
		Запуск в режиме панели процессора processorName
	    processorName может быть идентификатором требуемого процессора.
	/exp:exportJobName
		Экспорт данных. Если exportJobName является именем асинхронного
		кассового узла, то экспорт справочника товаров для асинхронной
		кассовой сессии.
		Другие задания для экспорта:
			MONARCH - экспорт данных для принтера штрих-кодов Monarch
			LP15    - экспорт данных для весов LP15
	/imp:importJobName
		Импорт данных. Если importJobName является именем асинхронного
		кассового узла, то импорт результатов работы асинхронной
		кассовой сессии.
	/in:path
		Позволяет переопределить путь доступа к входящим файлам
	/out:path
		Позволяет переопределить путь доступа к исходящим файлам
	/syncput
		Передача документов другому разделу БД
	/syncget
		Прием пакетов синхронизации из других разделов БД
	/backup
		Автоматическое резервное копирование в соответствии со сценариями
	/batch
		Пакетный режим работы. Позволяет выполнить работу, заданную
		командной строкой и немедленно выгрузить программу.
	/db:dbname
		Имя базы данных, с которой следует работать
	/goodsinfo:infokioskname
		Название информационного киоска, который следует запустить
	/verhist:dbname
		Выводит в файл историю смены версий для БД dbname
	/nologin
		Запускает программу без авторизации и входа в БД (усеченное меню)
--------------------------------------------------------------------------------
PPWS param 
param:
	install - установка сервиса Papyrus Server на компьютер
		Внимание! При запуски программы с этой опцией проследите за тем,
	 	чтобы в операционной системе путь запуска был привязан к локальному диску.
		Пример. Допустим Papyrus установлен в каталоге C:\WORK\PPY, который
		в сети вдиен как \\SERVER\DISK_C\WORK\PPY и подключен как
		H:\PPY (H: = \\SERVER\DISK_C\WORK). В этом случае установку сервиса
		следует осуществлять так: c:\work\ppy\bin\ppws install, 
		но не h:\ppy\bin\ppws install и не \\server\disk_c\work\ppy\bin\ppws install
	uninstall - деинсталляция сервиса Papyrus Server
	server - запуск программы в режиме серверного сервиса
